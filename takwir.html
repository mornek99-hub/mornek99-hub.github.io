AYAT 01 إِذَا الشَّمْسُ كُوِّرَتْ "Wenn die Sonne eingewickelt (gefaltet) wird."
COMPRESSION ALGORITHM: KUWWIRAT (Einwickeln/Rollen) beschreibt technisch das Zusammenfalten von Dimensionen. Die Lichtquelle der Simulation (Sonne/Display) wird ausgeschaltet und der Raum komprimiert (wie eine ZIP-Datei).
AYAT 02 وَإِذَا النُّجُومُ انكَدَرَتْ "Und wenn die Sterne herabstürzen (trübe werden)."
PIXEL LOSS / POWER DOWN: INKADARAT bedeutet "trübe werden" oder "Struktur verlieren". Die Hintergrund-Assets der Simulation (Sterne) verlieren ihre Auflösung und werden schwarz. Das System fährt herunter.
AYAT 03 وَإِذَا الْجِبَالُ سُيِّرَتْ "Und wenn die Berge bewegt werden."
STATIC OBJECT DELETION: Berge sind die "Anker" der Physik-Engine (statische Objekte). SUYYIRAT (in Bewegung setzen) bedeutet, dass diese festen Parameter gelöscht oder verschoben werden. Die physische Realität löst sich auf.
AYAT 07 وَإِذَا النُّفُوسُ زُوِّجَتْ "Und wenn die Seelen gepaart werden."
DATA MERGING: ZUWWIJAT (Paarung/Zusammenfügung) ist der Prozess, bei dem der User (Nafs) mit seinen gespeicherten Daten (Backup/Log-File) endgültig verschmolzen wird. Identität + Taten werden eins.
AYAT 10 وَإِذَا الصُّحُفُ نُشِرَتْ "Und wenn die Blätter (Seiten) ausgebreitet werden."
LOG FILE REVIEW: Die "Suhuf" (Blätter/Datenträger) sind die Logs, die wir in Sequenz 02 (Qalam) geschrieben haben. Jetzt werden sie "ausgebreitet" (entpackt/gelesen). Der Code wird validiert.
